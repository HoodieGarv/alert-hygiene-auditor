# Contributing

Contributions are welcome. To keep the review process straightforward, please follow the workflow described below.

Fork the repository and create a feature branch from `main` with a name that describes the change you intend to make. Make your changes on that branch, keeping commits focused and the scope of each pull request narrow. Before opening a pull request, run the test suite with `pytest` to confirm that all tests pass, and run `ruff check .` to confirm that the code is free of lint errors. Open a pull request against `main`, include a clear description of what the change does and why it is needed, and ensure that CI passes before requesting a review.
